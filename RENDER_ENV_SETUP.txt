# Render Environment Variables for Supabase

## Option 1: Direct Connection (try this first)
```
DB_CONNECTION=pgsql
DB_HOST=db.wlcuedmyoilkzqytskyi.supabase.co
DB_PORT=5432
DB_DATABASE=postgres
DB_USERNAME=postgres
DB_PASSWORD=carlDB@2025DB
DB_SSLMODE=require
```

## Option 2: Connection Pooler (if Option 1 has network issues)
If you get "Network is unreachable" error, try using Supabase's connection pooler:

Go to Supabase → Settings → Database → Connection Pooling
Look for the "Connection pooling" host, it should be like:
```
DB_CONNECTION=pgsql
DB_HOST=aws-0-ap-southeast-1.pooler.supabase.com
DB_PORT=6543
DB_DATABASE=postgres
DB_USERNAME=postgres.wlcuedmyoilkzqytskyi
DB_PASSWORD=carlDB@2025DB
DB_SSLMODE=require
```

Note: 
- Port changes from 5432 to 6543
- Username might include project ID (check Supabase dashboard)

## Steps to Apply:

1. Go to: https://dashboard.render.com
2. Select your `budget-tracker` service
3. Click **Environment** tab on the left
4. For each variable above:
   - If it exists: Click **Edit** and update the value
   - If it doesn't exist: Click **Add Environment Variable**
5. Add your actual Supabase password for `DB_PASSWORD`
6. Click **Save Changes**
7. Render will automatically redeploy with new database connection

## After Setting Environment Variables:

The app will automatically:
- Connect to your Supabase database
- Run migrations on startup (creates all tables)
- Be ready to use!

## Verify Setup:

1. Check Render logs for successful migration
2. Try registering a user on your app
3. Check Supabase Table Editor to see the data
